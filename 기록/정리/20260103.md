# 다이어리 멤버 초대 / 멤버 리스트 조회

- 멤버 초대

  - 멤버로 초대할 때 email을 알아야 하므로 email 받는 dto 하나 생성
  - 방장만 초대 가능하므로 방장 권한 확인
  - 개인 다이어리는 멤버 초대 불가
  - 초대 할 유저가 존재 하는지 / 이미 그룹 멤버인지 확인 후 멤버로 추가

- 멤버 리스트 조회
  - 다이어리가 존재하는 지 확인 후 해당 다이어리 그룹에 포함된 멤버 조회

# 다이어리 공지 수정 / 다이어리 그룹 삭제 및 탈퇴

- 공지 수정

  - 공지를 받는 request dto 생성해서 작성
  - 공지는 멤버면 전부 수정 가능하므로 따로 권한 체크는 하지 않음

- 다이어리 삭제 및 탈퇴
  - 삭제는 방장만 가능 -> 방장 권한 확인 후 그룹 삭제, 연관된 일기 멤버 전부 삭제
  - 탈퇴는 멤버만 가능 -> 방장은 탈퇴 불가능(추후에 방장 권한 위임 후 탈퇴 가능 기능 구현 할 지 생각 중)

## ResponseEntity.ok().build() 를 사용하는 이유

- HTTP 응답의 상태 코드(Status Code)와 본문(Body)을 명확하고 유연하게 제어하기 위함

- ResponseEntity.ok()는 HTTP 상태 코드 200 OK를 응답으로 보내겠다는 뜻

- ResponseEntity.ok().build()는 데이터 없이 성공만 알릴 때
- 데이터를 보낼 때는 ResponseEntity.ok(data)

- 빌더 패턴 (Builder Pattern) 사용
  ResponseEntity는 객체를 생성할 때 빌더 패턴을 지원함

  .ok(): 상태 코드는 200으로 설정한다는 뜻

  .build(): 더 이상 추가할 설정(헤더나 본문 등)이 없으니 이대로 ResponseEntity 객체를 완성해서 반환한다는 뜻
